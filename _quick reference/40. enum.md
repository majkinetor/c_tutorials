# C Enum Quick Reference

## Basic Syntax

Enums define a set of named integer constants. Without explicit values, they start at 0 and increment automatically.

```c
enum Day {
    MONDAY,      // 0
    TUESDAY,     // 1
    WEDNESDAY,   // 2
    THURSDAY,    // 3
    FRIDAY,      // 4
    SATURDAY,    // 5
    SUNDAY       // 6
};

enum Day today = WEDNESDAY;
```

> **Note:** Values start at 0 by default and increment by 1

## Explicit Values

Assign specific integer values to enum constants. Useful for error codes or status values.

```c
enum Status {
    ERROR = -1,
    SUCCESS = 0,
    PENDING = 1,
    COMPLETED = 2
};
```

## Using typedef

Combine enum declaration with typedef to create a cleaner type name, eliminating the need to write `enum` before every variable declaration.

```c
typedef enum {
    RED,
    GREEN,
    BLUE,
    ALPHA
} Color;

// Cleaner usage without 'enum' keyword
Color background = BLUE;
```

> **Note:** `typedef` eliminates need to write `enum` repeatedly

## Switch Statements

Enums work perfectly with switch statements, making code more readable and maintainable than using raw integers.

```c
switch (today) {
    case MONDAY:
    case TUESDAY:
    case WEDNESDAY:
    case THURSDAY:
    case FRIDAY:
        printf("Weekday\n");
        break;
    case SATURDAY:
    case SUNDAY:
        printf("Weekend\n");
        break;
}
```

## Partial Assignment

Mix automatic and explicit values. After an explicit assignment, subsequent values continue incrementing from that point.

```c
enum Values {
    FIRST = 10,
    SECOND,        // 11
    THIRD,         // 12
    RESET = 0,
    NEXT           // 1
};
```

> **Note:** Unspecified values continue from previous + 1


## Key Points

Important characteristics and limitations of C enums to keep in mind:

- Enums are integers under the hood
    - `sizeof(enum Type) == sizeof(int)`
- Improve code readability vs magic numbers
- Cannot have duplicate names in same scope
