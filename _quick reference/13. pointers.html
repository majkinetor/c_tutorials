C POINTERS QUICK REFERENCE

Declaration and Initialization:

int x = 42;
int *p = &x; // p holds address of x
0x1000
x: 42
←
0x2000
p: 0x1000

Dereferencing:

int value = *p; // read value at address
*p = 100; // write to address

Pointer Arithmetic:

int arr[] = {10, 20, 30, 40};
int *p = arr;
0x1000
10
0x1004
20
0x1008
30
0x100C
40
p++; // moves 4 bytes (sizeof(int))
p += 2; // moves 8 bytes (2 * sizeof(int))

Array Equivalence:

arr[2] ≡ *(arr + 2)
&arr[2] ≡ arr + 2

Common Pitfalls:

  • Dereferencing NULL/uninitialized
  • Using memory after free()
  • Buffer overruns
  • Confusing * in declaration vs use

NULL Pointer:

int *p = NULL; // always initialize!
if (p != NULL) { *p = 5; }

Void Pointer:

void *generic = &x; // any type
int *typed = (int *)generic; // cast needed
